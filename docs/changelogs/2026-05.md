# Changelog — May 2026

## 2026-05-19 — SSE response streaming

- **The turn response now streams over Server-Sent Events.** `/api/turn` switched
  from a buffered `messages.create` to `messages.stream`, emitting `delta` /
  `done` / `error` SSE frames; `runTurn` (`lib/api.ts`) consumes the stream and
  reports text as it arrives via an `onDelta` callback; the UI renders Sal's
  reply live in a streaming bubble with a blinking caret. This is still **one
  API call per turn** — `messages.stream` is a single request, just delivered
  incrementally — so the Phase 1.5 invariant holds.
- **Turn metadata moved from a fenced ```json block to a `<turn-meta>` …
  `</turn-meta>` tag pair.** The explicit delimiter lets the streaming UI hide
  the confidence-score block the instant its opening tag appears, and retires
  the old "anchor on the *last* fenced block" heuristic — a unique tag pair is
  unambiguous. `parseTurnResponse` and the prompt's OUTPUT FORMAT section were
  updated to match; `stripStreamingMeta` is the new mid-stream sibling that
  hides the block (and partial tags split across SSE chunks) from the live
  bubble.
- **Mid-stream metadata hiding is content-aware.** `stripStreamingMeta` treats a
  `<turn-meta>` as the metadata block only when it is followed by JSON (`{`), so
  a bare mention of the tag in Sal's prose stays visible — keeping the streaming
  bubble consistent with `parseTurnResponse`, which likewise only accepts a
  JSON-bearing block.
- **The client-disconnect abort hangs off the response, not the request.** A
  browser hang-up mid-turn aborts the upstream call, via `res.on('close')` + a
  `writableEnded` guard. It must be the *response*: `req`'s `'close'` fires
  within ~2 ms of the handler starting — as soon as the already-read POST body
  stream is destroyed — so a `req`-close abort would kill every normal turn.
  See the comment at the abort site in `src/server/index.ts`.

## 2026-05-18 — Dev config

- **Dev server port 5173 → 5555.** Moved the Vite dev client off the crowded
  default to avoid collisions, and added `strictPort: true` so a taken port now
  fails loudly instead of silently drifting to 5174 (a drifted port breaks the
  server's `CORS_ORIGIN` match). Updated in `vite.config.ts`, the server's
  `CORS_ORIGIN` default, `.env.example`, and the docs.
- **`ANTHROPIC_MAX_TOKENS` default 4096 → 16384.** At 4096 a long turn could be
  truncated *before* Sal's trailing JSON metadata block — `parseTurnResponse`
  then finds no block, returns `metadata: null`, and confidence scores silently
  stop updating. The higher ceiling gives the reply + metadata headroom.

## 2026-05-18 — Review fixes

- **parseTurnResponse: parse the last metadata block, not the first.** It
  matched the first fenced ```json block — so a JSON example in Sal's answer
  was mis-parsed as metadata and truncated the visible reply. Now anchors on
  the final block, requires it to sit at end-of-response and carry a
  `confidence_scores` object, and leaves the text intact otherwise. Added
  `lib/prompt.test.ts` with a regression test for the case.
- **Server: presence-based static serving.** `dist/client` is now served when
  it exists rather than when `NODE_ENV === 'production'`, so `npm start` serves
  the built UI without an env flag (and stays inert in dev).
- **Server: explicit missing-key handling.** With no `ANTHROPIC_API_KEY` the
  server starts, warns on boot, and `/api/turn` returns a clear 500 setup error
  instead of a generic one (the SDK's missing-key error is not an `APIError`).
- **health-check.sh: diagnose platform/`node_modules` mismatch.** A Rollup/
  esbuild native-module load failure is now reported as an environment
  mismatch, not a test failure — avoids false QA failures across shells.
- **npm audit: bumped `@anthropic-ai/sdk` to ^0.96.0**, clearing one moderate
  advisory. The remaining dev-only esbuild/vite/vitest advisories are deferred
  to a deliberate Vite/Vitest major upgrade rather than force-fixed.

## 2026-05-18 — Package spinup

- **Build tooling.** Stood up a proper TypeScript project: Vite 6 + React 19
  client, Vitest test harness, ESLint, and `tsconfig` split (client / server).
  Conventions adapted from the Miles-Chat repo.
- **Restructure.** Converted the single-file `sgc-phase-1-5.jsx` artifact into
  `src/`: the pure TF-IDF engine (`lib/tfidf.ts`), prompt builder/parser
  (`lib/prompt.ts`), transport (`lib/api.ts`), shared types, and the React UI
  (`SalienceGatedCognition.tsx`). The original is frozen at
  `docs/phase-1-5-reference.jsx`. Dropped dead `currentPhase` state.
- **Turn proxy.** Added `src/server/index.ts` — a thin Express server with one
  route (`POST /api/turn`) that holds `ANTHROPIC_API_KEY` and calls
  `@anthropic-ai/sdk`. This replaces the artifact's headerless browser fetch to
  `api.anthropic.com`. Model defaults to `claude-opus-4-7`.
- **Tests.** Added `lib/tfidf.test.ts` — behavioral coverage for the TF-IDF
  cosine engine (tokenize, TF/IDF, cosine similarity, search).
- **Naming.** Settled the model's identity as **Sal** (used wherever a user
  sees it) and adopted **turn** as the neutral word for the mechanism
  (`runTurn`, `/api/turn`, `TurnData`, `TurnMetadata`). Retired the "Synth"
  working title from the live code and docs; it survives only in the frozen
  reference artifact.
