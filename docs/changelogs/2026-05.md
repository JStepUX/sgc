# Changelog — May 2026

## 2026-05-19 — Chat body: markdown rendering + lighter body weight

- **Assistant replies now render as markdown** via `react-markdown` +
  `remark-gfm` (same combo Miles-Chat uses). Lists, bold, italic, inline and
  fenced code, blockquotes, headings, tables, and hrs all map onto Sal palette
  tokens — bold is `font-medium text-fg-1` (not 600/700, which would shout on
  the warm field), inline code sits on `bg-surface-strong` in Geist Mono,
  blockquotes use `text-fg-2` with a `border-hairline-strong` rail, links pick
  up `text-ember-soft` with an ember underline. Previously a `- **foo**` from
  Sal rendered as the literal characters; now it's a bulleted list with a bold
  first word.
- **Chat body copy dropped to Geist 300 (`font-light`).** Both `AssistantMessage`
  prose (15px) and `UserPill` text (14.5px) were at Geist 400, which on a warm
  near-black field optically blooms a half-weight heavier than its dark-on-light
  equivalent. The rails, memory cards, phase bar, and all mono labels keep
  their existing weights — the bump is chat-only. Geist 300 is already loaded
  via the `<link>` in `index.html`, so no new font request.
- **Streaming cursor moved from a manual `<span>` at the end of the last
  paragraph to a `::after` pseudo-element** on `.sal-streaming > *:last-child`
  (CSS in `index.css`). Same thin ember bar, same `sal-blink` animation — but
  it now attaches naturally to whatever the last markdown block ends up being
  (a `<p>`, a `<li>`, a heading), instead of needing the component to manually
  split paragraphs and remember which one is last. The pre-markdown component
  split the text on `\n{2,}` and only set the cursor on the final chunk; the
  pseudo-element does the same job structurally.
- **Phase 1.5 invariants untouched.** No new API call, no model-based
  retrieval, no change to `lib/` (TF-IDF, prompt builder, SSE transport) or
  the server. Pure presentation change in `SalienceGatedCognition.tsx` +
  `index.css`. New deps: `react-markdown@^10.1.0`, `remark-gfm@^4.0.1`. 34/34
  Vitest tests still pass; lint + typecheck clean; visually verified
  end-to-end at 1280-wide via Playwright MCP — bullets render with disc
  markers, `<strong>` computes to weight 500, the message wrapper computes
  to weight 300.

## 2026-05-19 — UI re-skin: Sal design system; Tailwind v4 + shadcn/ui

- **Adopted Tailwind v4 + shadcn/ui** as the styling stack. The
  `@tailwindcss/vite` plugin processes a single `src/client/index.css` that
  holds the Tailwind import, the Sal palette in an `@theme` block (warm earthy
  tokens — `--color-ember`, `--color-fg-1..4`, `--color-hairline*`, aurora
  colours), the shadcn dark theme in `:root` + `@theme inline`, and the aurora
  CSS / keyframes (intrinsically CSS, not utility-shaped). shadcn primitives
  `Button` and `Card` live under `src/client/components/ui/`; `cn()` in
  `src/client/lib/utils.ts`; `components.json` configures the shadcn CLI. New
  deps: `tailwindcss`, `@tailwindcss/vite`, `clsx`, `tailwind-merge`,
  `class-variance-authority`, `lucide-react`, `@radix-ui/react-slot`. The
  `no Tailwind` line in CLAUDE.md is gone.
- **Re-skinned the UI to the Sal design system** — a warm near-black field,
  aurora gradients that drift only while the user is active, a per-keystroke
  ember pulse, ember as the single saturation accent. Geist + Geist Mono via
  `<link>` in `index.html`. The phase bar, the thread (now a centred reading
  column, not chat bubbles), the auto-growing composer, and the context rail
  (constitutional memories, architecture trace, grep citations, API-call card,
  confidence deltas, token chart) all redrawn. `SalienceGatedCognition.tsx`
  dropped from 1040 lines (380 of which were an embedded CSS string) to 684
  lines of actual React. Brand voice applied — "Say something." placeholder,
  "Begin again" reset, "considering" while reasoning, "I lost my place. Try
  again?" on error.
- **Aurora uses CSS transitions, not a per-frame RAF.** An earlier draft of
  the re-skin ran `requestAnimationFrame` forever calling `setGate`, easing
  asymptotically toward a target that was never reached — re-rendering the
  app tree at ~60fps even while idle. The element already carries
  `transition: filter 600ms` in `index.css`; the gate is now derived directly
  from the input on each render, and the browser does the easing in
  compositor. Re-renders are O(keystroke), not O(frame).
- **Narrow widths stack instead of breaking.** Below `lg` (1024px) the
  context rail moves below the thread (capped at `45vh`); below `md` (768px)
  the phase bar's meta strip (`1 API call/turn · TF-IDF Grep · 2-turn
  buffer`) hides, leaving the title, PHASE badge, and the `Begin again`
  reset. Phone is still not a target — the rail isn't a drawer, just a
  stacked panel.
- **Phase 1.5 invariants untouched.** Pure visual + presentation change.
  `lib/` (TF-IDF, prompt builder, SSE transport), the Express server, and
  the one-API-call-per-turn / no-model-retrieval contract are byte-for-byte
  unchanged. 34/34 Vitest tests pass; visual smoke verified at 1280 / 900 /
  680 widths.

## 2026-05-19 — Confidence-delta panel doubling

- **The Turn Inspector's Confidence Deltas list showed each memory twice**
  (M1–M6 for three memories). `processInput` built the delta array by
  `push`-ing from *inside* the `setMemories` updater — an impure updater, which
  React `<StrictMode>` double-invokes in dev, doubling the array. The delta
  computation now runs once, up front, as a plain value; the `setMemories`
  updater is pure (maps `prev` and returns). Dev-only and display-only —
  memory confidence values were always correct — but the impure updater was a
  latent hazard. Found during the SSE streaming QA pass.

## 2026-05-19 — SSE response streaming

- **The turn response now streams over Server-Sent Events.** `/api/turn` switched
  from a buffered `messages.create` to `messages.stream`, emitting `delta` /
  `done` / `error` SSE frames; `runTurn` (`lib/api.ts`) consumes the stream and
  reports text as it arrives via an `onDelta` callback; the UI renders Sal's
  reply live in a streaming bubble with a blinking caret. This is still **one
  API call per turn** — `messages.stream` is a single request, just delivered
  incrementally — so the Phase 1.5 invariant holds.
- **Turn metadata moved from a fenced ```json block to a `<turn-meta>` …
  `</turn-meta>` tag pair.** The explicit delimiter lets the streaming UI hide
  the confidence-score block the instant its opening tag appears, and retires
  the old "anchor on the *last* fenced block" heuristic — a unique tag pair is
  unambiguous. `parseTurnResponse` and the prompt's OUTPUT FORMAT section were
  updated to match; `stripStreamingMeta` is the new mid-stream sibling that
  hides the block (and partial tags split across SSE chunks) from the live
  bubble.
- **Mid-stream metadata hiding is content-aware.** `stripStreamingMeta` treats a
  `<turn-meta>` as the metadata block only when it is followed by JSON (`{`), so
  a bare mention of the tag in Sal's prose stays visible — keeping the streaming
  bubble consistent with `parseTurnResponse`, which likewise only accepts a
  JSON-bearing block.
- **The client-disconnect abort hangs off the response, not the request.** A
  browser hang-up mid-turn aborts the upstream call, via `res.on('close')` + a
  `writableEnded` guard. It must be the *response*: `req`'s `'close'` fires
  within ~2 ms of the handler starting — as soon as the already-read POST body
  stream is destroyed — so a `req`-close abort would kill every normal turn.
  See the comment at the abort site in `src/server/index.ts`.

## 2026-05-18 — Dev config

- **Dev server port 5173 → 5555.** Moved the Vite dev client off the crowded
  default to avoid collisions, and added `strictPort: true` so a taken port now
  fails loudly instead of silently drifting to 5174 (a drifted port breaks the
  server's `CORS_ORIGIN` match). Updated in `vite.config.ts`, the server's
  `CORS_ORIGIN` default, `.env.example`, and the docs.
- **`ANTHROPIC_MAX_TOKENS` default 4096 → 16384.** At 4096 a long turn could be
  truncated *before* Sal's trailing JSON metadata block — `parseTurnResponse`
  then finds no block, returns `metadata: null`, and confidence scores silently
  stop updating. The higher ceiling gives the reply + metadata headroom.

## 2026-05-18 — Review fixes

- **parseTurnResponse: parse the last metadata block, not the first.** It
  matched the first fenced ```json block — so a JSON example in Sal's answer
  was mis-parsed as metadata and truncated the visible reply. Now anchors on
  the final block, requires it to sit at end-of-response and carry a
  `confidence_scores` object, and leaves the text intact otherwise. Added
  `lib/prompt.test.ts` with a regression test for the case.
- **Server: presence-based static serving.** `dist/client` is now served when
  it exists rather than when `NODE_ENV === 'production'`, so `npm start` serves
  the built UI without an env flag (and stays inert in dev).
- **Server: explicit missing-key handling.** With no `ANTHROPIC_API_KEY` the
  server starts, warns on boot, and `/api/turn` returns a clear 500 setup error
  instead of a generic one (the SDK's missing-key error is not an `APIError`).
- **health-check.sh: diagnose platform/`node_modules` mismatch.** A Rollup/
  esbuild native-module load failure is now reported as an environment
  mismatch, not a test failure — avoids false QA failures across shells.
- **npm audit: bumped `@anthropic-ai/sdk` to ^0.96.0**, clearing one moderate
  advisory. The remaining dev-only esbuild/vite/vitest advisories are deferred
  to a deliberate Vite/Vitest major upgrade rather than force-fixed.
