# 2026-06

## 2026-06-01 — Constitutional memories go per-chat (and stop shipping defaults)

Constitutional memories were **global** — one shared set across every chat — and
the app seeded three hardcoded defaults so a fresh install "had something to push
on." Both are now gone: memories are **scoped per chat** and **start empty**.

- **Data model.** `memories` gains a `chat_id` FK (`ON DELETE CASCADE`), so each
  chat owns its own constitutional set and deleting a chat drops its memories
  with it. The schema DDL is hoisted into a `MEMORIES_SCHEMA_SQL` const shared by
  the fresh-create path and a one-time migration: a legacy table (no `chat_id`)
  is dropped and rebuilt — the old global rows belong to no chat, so they're
  discarded by design (confirmed acceptable for the local research-prototype DB).
  Every memory statement (`getMemories`, `saveMemories`, the reconciliation
  id-sweep, the upsert) is now scoped to `chat_id`; critically the save's
  delete-absent sweep only touches *this* chat's rows, never another chat's.
- **Transport.** A chat's memories ride along in its `loadChat`
  (`GET /api/chats/:id`) payload now — the standalone `GET /api/memories` route
  and the client `getMemories()` are removed (one load hydrates turns + persona +
  memories together). `PUT /api/memories` carries a required `chatId`.
- **Client.** `DEFAULT_MEMORIES` is deleted; `memories` initialises to `[]`.
  Hydration and chat-switch load the active chat's set from the detail and reset
  the dirty ref so the programmatic load isn't mistaken for an edit; "Begin again"
  empties the set; the debounced save now keys on the active `chatId`. Net effect:
  add memories in chat A, switch to B → B is empty, switch back → A's are intact
  and distinct.
- **Prompt.** With an empty set the `CONSTITUTIONAL MEMORIES:` block renders
  `(none yet …)` instead of a blank section under the "you carry constitutional
  memories" framing — so the prompt never claims memories that don't exist. New
  pinned test covers the empty case; the non-empty path is byte-identical.
- Still inside the Phase 1.5 invariants: pure curation, no model in the memory
  path, no new API call — only the *scope* and *seed* of the tier changed.
- **Review hardening.** Four follow-ups from a critical review:
  - *No lost edits on chat swap.* The 250ms save debounce was cancelled when a
    chat switch / "Begin again" replaced the outgoing set within the window —
    silently dropping the edit (a regression from the per-chat swap). The swap
    callbacks now `flushPendingMemorySave()` first (reads live refs), and the
    debounce clears the dirty flag as it fires so `dirty` means "unsaved
    changes," keeping the flush a no-op when nothing is pending.
  - *Cross-chat id guard.* `saveMemories` now rejects (`409`) an incoming memory
    id already owned by a different chat. Can't happen via the UI (UUIDs, never
    copied), but the id-keyed upsert would otherwise silently rebind the owner's
    row — defense made loud rather than a `(chat_id, id)` schema rework.
  - *Missing-chat saves are `404`, not `500`.* The PUT now surfaces "chat not
    found" (a check in `saveMemories`, mirroring `saveTurnPair`), which also
    fixes an empty payload to a deleted chat silently returning `200`.
  - Stale "memories are global" comment on `startNewChat` corrected.

## 2026-06-01

- **Brain surgery — manual timeless memories.** The chat memory editor can now
  *insert* turns, not just gate them. An "Add memory" button (top-right, beside
  the close X) opens a form capturing a full user+assistant turn; on submit the
  pair is inserted as the **oldest** turn in the chat and flagged `timeless`.
  Timeless turns are greppable like any other (still pure TF-IDF, no model) but
  the time scorer **negates recency** for them — `timeScore` is pinned to 1.0
  regardless of age or any time phrase in the query, so a curated fact ranks on
  concept alone and never decays. The cosine cliff still holds (concept 0 → no
  retrieval), so a timeless flag can't surface an off-topic memory. In the
  prompt these read `[Turn N · timeless]` instead of a relative-time tag.
  - **Data model.** New `turns.timeless` column (additive migration, default 0).
    `prependManualTurnPair` inserts below the minimum ordinal (negatives are
    fine — ordinals only sort) and does NOT bump `updated_at` (a manual memory
    is the *oldest* content, not new activity, so the chat shouldn't jump the
    history list). `deleteManualTurnPair` removes both halves of a pair to keep
    the user/assistant alternation the cosine engine relies on, and refuses any
    non-timeless turn. New routes: `POST /api/chats/:id/manual-turns`,
    `DELETE /api/chats/:id/turns/:turnId`.
  - **UI.** Timeless cards render with an ember gradient fill, a `timeless` tag,
    and a delete (trash) control in place of the on/off toggle; they're excluded
    from gating and mass actions (always retrievable).
- **Invariant enforced at the mutation, not just the UI.** `setTurnsActive`'s
  UPDATE now carries `AND timeless = 0`, so gating a timeless turn off is a
  silent no-op even if an API caller or future code path hands it a timeless id
  — the "manual memories are always retrievable" promise no longer relies on the
  UI omitting a toggle. (Core value: build the check, don't trust the self-report.)
- **Fixes from review.** Adding/deleting a manual memory now also refreshes the
  chat summaries (`handleTurnsMutated`) — without it, inserting the first turn
  into a previously-empty chat left a stale title/snippet in the history list.
  The Add Memory form now contains keyboard focus to its own sheet (a form-local
  trap) so Tab can't reach the behind-overlay chrome, and the dialog focus-trap
  selector now includes `textarea`/`select`.
