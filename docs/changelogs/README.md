# Changelogs

One file per month, named `YYYY-MM.md` (e.g. `2026-05.md`). Every committed
change gets an entry — the `pre-commit-qa` skill enforces this.

## Convention

- Newest entries at the top of the file.
- Each entry is a dated bullet under a `## YYYY-MM-DD` heading.
- Write for a developer reading it six months from now: be specific about *what*
  changed and *why*, not just *that* something changed. No boilerplate.

```
## 2026-05-18

- Short, specific description of the change and its motivation.
```
